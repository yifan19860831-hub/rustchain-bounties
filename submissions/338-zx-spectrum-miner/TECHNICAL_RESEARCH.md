# ZX Spectrum Miner - 技术架构研究

## 1. ZX Spectrum 硬件规格 (1982)

### 核心硬件
- **CPU**: Zilog Z80A @ 3.5 MHz (NTSC) / 3.25 MHz (PAL)
- **架构**: 8 位
- **寄存器**: 
  - 8 个 8 位通用寄存器 (A, B, C, D, E, H, L) + 镜像组
  - 4 个 16 位寄存器对 (AF, BC, DE, HL)
  - 程序计数器 PC (16 位)
  - 堆栈指针 SP (16 位)
  - 索引寄存器 IX, IY (16 位)
- **RAM**: 16 KB / 48 KB (标准配置)
- **ROM**: 16 KB (包含 BASIC 解释器和系统例程)

### 内存映射
```
0x0000 - 0x3FFF: ROM (16 KB)
0x4000 - 0x7FFF: RAM (48 KB 模式) 或 空 (16 KB 模式)
0x8000 - 0xFFFF: RAM (仅 48 KB 模式)
```

### 显示
- 分辨率：256 x 192 像素
- 颜色：15 色 (2 色 per 8x8 字符块)
- 显存：6912 字节 (0x4000 - 0x57FF)

### 存储
- 原始：盒式磁带 (Cassette)
- 加载速度：~1500 bps (极慢!)
- 后期：+3 磁盘驱动器 (3" 软盘)

### 网络
- **无内置网络支持**
- 需要外部硬件扩展 (如 ZX Interface 2, 或现代扩展)

## 2. 移植可行性分析

### 限制因素
| 因素 | ZX Spectrum | 现代矿工需求 | 差距 |
|------|-------------|--------------|------|
| RAM | 48 KB | GB 级 | 100,000x |
| 存储 | 磁带/软盘 | SSD/NVMe | 无法比较 |
| 网络 | 无 | 必需 | 需要扩展 |
| 加密原语 | 无硬件支持 | SHA-256 等 | 软件实现极慢 |
| 计算能力 | 0.00035 GFLOPS | TH/s | 10^15x |

### 结论
**直接在真实硬件上运行完整矿工 = 不可能**

### 可行方案：概念验证/教育性移植

#### 方案 A: Z80 汇编 PoC (推荐)
- 实现极简的"矿工"逻辑
- 使用预定义难度目标
- 在模拟器上运行和演示
- 重点展示原理，而非实际挖矿

#### 方案 B: Python 模拟器
- 用 Python 模拟 Z80 CPU
- 在模拟器上运行 Z80 矿工代码
- 可连接到测试网络
- 教育价值高

#### 方案 C: 混合方案
- Z80 代码处理核心逻辑
- Python 处理网络和存储
- 通过模拟 I/O 端口通信

## 3. 极简矿工设计

### 核心功能 (简化版)
```
1. 获取区块头 (简化为固定数据)
2. 递增 nonce (16 位计数器)
3. 计算哈希 (简化版，非完整 SHA-256)
4. 检查是否满足难度目标
5. 显示结果
```

### 内存布局
```
0x8000 - 0x80FF: 区块头数据 (256 字节)
0x8100 - 0x8101: Nonce 计数器 (16 位)
0x8102 - 0x81FF: 哈希结果缓冲区 (254 字节)
0x8200 - 0x82FF: 工作栈
```

### Z80 汇编伪代码
```asm
ORG 0x8000

START:
    LD DE, 0x8100      ; Nonce 地址
    LD HL, 0           ; 初始 nonce = 0
    
MINING_LOOP:
    ; 存储 nonce
    LD (DE), HL
    
    ; 调用哈希函数 (简化)
    CALL HASH_FUNCTION
    
    ; 检查结果
    CALL CHECK_DIFFICULTY
    JR Z, FOUND_BLOCK  ; 如果找到，跳转
    
    ; nonce++
    INC HL
    JR MINING_LOOP
    
FOUND_BLOCK:
    ; 显示成功
    CALL DISPLAY_SUCCESS
    RET
```
