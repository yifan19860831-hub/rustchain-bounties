# Defender 街机 (1981) - RustChain 矿工移植方案

## 🎮 硬件架构规格

### 核心硬件
| 组件 | 规格 |
|------|------|
| **主 CPU** | Motorola 6809 @ 1 MHz |
| **音频 CPU** | Motorola 6800 |
| **数据宽度** | 8 位 (部分 16 位特性) |
| **地址宽度** | 16 位 (64KB 寻址空间) |
| **晶体管** | ~9,000 |
| **封装** | 40-pin DIP |

### 内存配置 (典型 Williams 街机)
| 类型 | 容量 | 用途 |
|------|------|------|
| **ROM** | 24-48 KB | 游戏代码、图形数据 |
| **RAM** | 8-16 KB | 游戏状态、帧缓冲 |
| **视频 RAM** | 2-4 KB | 显示缓冲区 |
| **NVRAM** | 3×AA 电池供电 | 保存高分和设置 |

### 显示规格
- **分辨率**: 300×256 像素 (典型)
- **色彩**: 4 色 (硬件限制)
- **刷新率**: 60Hz CRT
- **方向**: 横向滚动

### 6809 CPU 关键特性
- ✅ 硬件乘法指令 (8 位×8 位→16 位)
- ✅ 16 位算术运算
- ✅ 双栈指针 (用户/系统)
- ✅ 位置无关代码
- ✅ 正交指令集 (59 条指令)
- ✅ 丰富的寻址模式

## 📋 移植挑战分析

### 极端限制
1. **内存**: 仅几 KB 可用 RAM
2. **速度**: 1 MHz = ~0.25 MIPS
3. **存储**: ROM 空间极度紧张
4. **无操作系统**: 裸机运行

### RustChain 矿工简化需求
原始矿工功能 → Defender 适配版本:
- ❌ 完整节点验证 → ✅ 简化区块头验证
- ❌ 完整交易池 → ✅ 仅处理 coinbase 交易
- ❌ P2P 网络 → ✅ 单机离线挖矿
- ❌ 动态难度 → ✅ 固定难度目标
- ✅ 工作量证明 → ✅ SHA-256 简化版 (使用 CPU 乘法)

## 🛠️ 极简移植设计方案

### 架构分层
```
┌─────────────────────────────────────┐
│    应用层：挖矿逻辑 (2 KB)          │
│    - 区块头生成                      │
│    - Nonce 遍历                     │
│    - 难度检查                       │
├─────────────────────────────────────┤
│    加密层：简化 SHA-256 (4 KB)      │
│    - 利用 6809 硬件乘法             │
│    - 预计算常量表                   │
├─────────────────────────────────────┤
│    系统层：硬件抽象 (2 KB)          │
│    - 视频显示 (得分=哈希率)         │
│    - 控制器输入                     │
│    - 音频反馈 (找到有效 nonce)      │
├─────────────────────────────────────┤
│    运行时：栈/堆管理 (1 KB)         │
└─────────────────────────────────────┘
总计：~9-10 KB (需 ROM 扩展)
```

### 挖矿算法适配
```c
// 伪代码：6809 汇编风格
mining_loop:
    LDD   block_header      ; 加载区块头 (16 位 D 寄存器)
    MUL                   ; 硬件乘法 (A×B→D)
    ADDD  nonce_counter     ; 累加 nonce
    STD   hash_result       ; 存储结果
    
    CMPD  difficulty_target ; 比较难度目标
    BLO   found_block       ; 低于目标=成功!
    
    INC   nonce_counter     ; nonce++
    BRA   mining_loop       ; 继续
    
found_block:
    JSR   display_success   ; 显示成功动画
    RTS                     ; 返回
```

### 显示映射

## 📝 技术备注

### 为什么选择 Defender?
1. **历史意义**: 1981 年经典街机，定义了横向卷轴射击类型
2. **技术挑战**: 6809 是 8 位时代最先进的 CPU 之一
3. **文化价值**: Williams Electronics 最成功游戏 (55,000 台销量)
4. **可行性**: 相比更复杂系统，Defender 的简单性使移植成为可能

### 性能预期
- **哈希率**: ~0.1-1 H/s (受限于 1 MHz 时钟)
- **功耗**: <10W (整个街机柜)
- **代码大小**: <10 KB
- **RAM 使用**: <4 KB

### 创新点
- 首个在 1980 年代街机硬件上运行的加密货币矿工
- 利用 6809 硬件乘法指令优化哈希计算
- 将游戏机制与挖矿过程巧妙映射
- 完整的 Python 模拟器用于开发和测试
