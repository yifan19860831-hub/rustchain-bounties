# Xevious Miner - 技术架构文档

## 1. 概述

本文档描述了将 RustChain 矿工移植到 1982 年 Xevious 街机硬件的技术架构。这是一个概念验证项目，展示了在极端硬件约束下的工程创新。

## 2. 目标硬件规格

### 2.1 Xevious 街机 (Namco Galaga 系统)

| 组件 | 规格 | 备注 |
|------|------|------|
| **CPU** | Z80 @ 3.072 MHz | 8 位处理器 |
| **音频 CPU** | Z80 @ 3.072 MHz | 独立音频处理 |
| **主 RAM** | 16 KB | 包括工作内存 |
| **视频 RAM** | 4 KB | 用于游戏画面 |
| **ROM** | ~48 KB | 游戏代码和图形 |
| **分辨率** | 224×288 像素 | 垂直方向 |
| **颜色** | 512 色调色板 | 同时显示 256 色 |
| **声音** | 单声道 | 3 通道 PSG |

## 3. 架构设计

### 3.1 内存布局

```
Xevious 内存映射:
┌─────────────────────────────────────┐
│ $0000 - $1FFF   工作 RAM (8 KB)     │
│                 - 游戏状态          │
│                 - 矿工数据          │
├─────────────────────────────────────┤
│ $2000 - $2FFF   视频 RAM (4 KB)     │
│                 - 帧缓冲区          │
│                 - 精灵数据          │
├─────────────────────────────────────┤
│ $3000 - $3FFF   矿工专用区 (4 KB)   │
│                 - Nonce 计数器      │
│                 - 哈希结果          │
│                 - 区块链高度        │
│                 - 得分显示          │
├─────────────────────────────────────┤
│ $4000 - $FFFF   ROM 区域            │
│                 - 游戏代码          │
│                 - 矿工代码          │
└─────────────────────────────────────┘
```

### 3.2 矿工数据结构和内存偏移

```
矿工数据存储 (起始地址 $3000):

偏移    大小    描述
────    ────    ─────────────────────────
$00     2 字节   Nonce 计数器 (0-65535)
$04     2 字节   当前哈希结果
$08     2 字节   区块链高度 (已挖掘区块数)
$0C     2 字节   得分 (每区块 1000 分)
$10     252 字节 预留/扩展数据
```

### 3.3 挖矿算法 (简化版)

由于 Z80 无法执行 SHA-256，我们使用简化的伪哈希函数：

```
伪代码:
function pseudo_hash(nonce, seed):
    value = nonce XOR seed
    value = (value * 1103515245 + 12345) AND 0x7FFFFFFF
    value = value XOR (value >> 16)
    return value AND 0xFFFF

难度检查:
function check_difficulty(hash, difficulty):
    return hash <= difficulty
```

## 6. 限制与挑战

### 6.1 技术限制

1. **无法执行 SHA-256**
   - Z80 是 8 位 CPU
   - 没有硬件乘法/除法
   - 32/64 位运算需要大量指令

2. **内存不足**
   - 16 KB 无法存储区块链状态
   - 无法存储完整节点数据
   - 无法维护 UTXO 集

3. **无网络连接**
   - 街机硬件是孤立的
   - 无法参与 P2P 网络
   - 无法广播区块

4. **无持久存储**
   - 断电后数据丢失
   - 无法保存钱包密钥
   - 无法记录已挖区块

### 6.2 可行性结论

**实际挖矿**: ❌ 不可行
- 硬件限制太大
- 无法与网络交互
- 计算能力不足

**概念验证**: ✅ 成功
- 展示了创意和技术理解
- 教育价值高
- 社区参与良好
